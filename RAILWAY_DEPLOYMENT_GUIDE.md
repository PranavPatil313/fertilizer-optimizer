# Fertilizer Optimizer - Railway Deployment Guide

This guide provides step-by-step instructions to deploy the Fertilizer Optimizer application on Railway.app.

## Prerequisites

1. **GitHub Account** - Your code must be in a GitHub repository
2. **Railway Account** - Sign up at [railway.app](https://railway.app) (free tier available)
3. **Vercel Account** (optional) - For frontend deployment (recommended)

## Deployment Options

### Option 1: Full Stack on Railway (Backend + Frontend)

#### Step 1: Deploy Backend Service

1. **Create New Project on Railway**
   - Go to [Railway.app](https://railway.app)
   - Click "New Project" → "Deploy from GitHub repo"
   - Select your `fertilizer-optimizer` repository
   - Railway will automatically detect the `Dockerfile` and deploy

2. **Add PostgreSQL Database**
   - In your Railway project dashboard, click "New" → "Database" → "PostgreSQL"
   - Railway will automatically create a `DATABASE_URL` environment variable

3. **Configure Environment Variables**
   - Go to your backend service → "Variables" tab
   - Add the following required variables:

   ```env
   # JWT Authentication
   JWT_SECRET=generate_a_strong_random_string_min_32_chars
   JWT_ALGORITHM=HS256
   ACCESS_TOKEN_EXPIRE_MINUTES=1440
   REFRESH_TOKEN_SECRET=another_strong_random_string
   REFRESH_TOKEN_EXPIRE_DAYS=7
   
   # API Security
   API_KEY=your_production_api_key_here
   
   # CORS Configuration (update with your frontend URL)
   CORS_ORIGINS=https://your-frontend.vercel.app,https://your-custom-domain.com
   
   # Optional: Weather API
   WEATHER_API_KEY=your_weather_api_key_here
   
   # Admin Configuration
   ADMIN_EMAILS=admin@example.com
   
   # ML Model Paths
   MODEL_PATH=src/artifact/model.pkl
   PREPROCESSOR_PATH=src/artifact/preprocessor.pkl
   ```

4. **Run Database Migrations**
   - After deployment, run migrations using Railway CLI:
   ```bash
   railway run alembic upgrade head
   ```
   - Or use the Railway dashboard: Service → "Shell" → run `alembic upgrade head`

#### Step 2: Deploy Frontend Service on Railway

1. **Create Separate Service for Frontend**
   - In your Railway project, click "New" → "Service"
   - Select "Deploy from GitHub repo"
   - Choose the same repository
   - Set the root directory to `fert-frontend`

2. **Configure Frontend Build Settings**
   - Build Command: `npm run build`
   - Output Directory: `dist`
   - Environment: Node.js

3. **Set Frontend Environment Variables**
   ```env
   VITE_API_BASE_URL=https://your-backend.railway.app
   VITE_API_KEY=your_production_api_key_here (must match backend API_KEY)
   VITE_APP_NAME=Fertilizer Optimizer
   ```

### Option 2: Backend on Railway + Frontend on Vercel (Recommended)

#### Step 1: Deploy Backend on Railway
Follow the same backend deployment steps as Option 1.

#### Step 2: Deploy Frontend on Vercel
1. Go to [vercel.com](https://vercel.com)
2. Import your GitHub repository
3. Configure project:
   - Framework: "Vite"
   - Root Directory: `fert-frontend`
   - Build Command: `npm run build`
   - Output Directory: `dist`
4. Add environment variables:
   - `VITE_API_BASE_URL`: Your Railway backend URL (e.g., `https://fertilizer-optimizer-production.up.railway.app`)
   - `VITE_API_KEY`: Same API key as backend

#### Step 3: Update CORS Configuration
- Go to Railway dashboard → Backend service → Variables
- Update `CORS_ORIGINS` to include your Vercel frontend URL:
  ```
  https://your-frontend.vercel.app,https://your-backend.railway.app
  ```

## Environment Variables Reference

### Required Backend Variables
| Variable | Description | Example |
|----------|-------------|---------|
| `DATABASE_URL` | Auto-generated by Railway PostgreSQL | `postgresql://...` |
| `JWT_SECRET` | Secret for JWT token signing | Random 32+ char string |

| `ADMIN_EMAILS` | Comma-separated admin emails | `admin@example.com` |

| Variable | Description | Example |
|----------|-------------|---------|

## Troubleshooting

### Common Issues

1. **Database Connection Failed**
   - Verify `DATABASE_URL` is set correctly
   - Check if PostgreSQL service is running
   - Run migrations: `railway run alembic upgrade head`

2. **Asyncpg Driver Error** (Common on Railway)
   - **Error**: `sqlalchemy.exc.InvalidRequestError: The asyncio extension requires an async driver to be used. The loaded 'psycopg2' is not async.`
   - **Cause**: Railway provides DATABASE_URL as `postgresql://` without `+asyncpg` driver specification
   - **Solution**: The application automatically converts `postgresql://` to `postgresql+asyncpg://`. If issue persists:
     - Manually set DATABASE_URL in Railway variables to include `+asyncpg` driver
     - Example: Change `postgresql://user:pass@host/db` to `postgresql+asyncpg://user:pass@host/db`
     - Restart the backend service

3. **Backend Won't Start**
   - Check Railway logs: Project → "Logs" tab
   - Verify all required environment variables are set
   - Check Docker build logs for errors

4. **CORS Errors**
   - Ensure `CORS_ORIGINS` includes your frontend domain
   - Restart backend after updating CORS configuration

5. **Frontend Can't Connect to Backend**
   - Verify `VITE_API_BASE_URL` is correct
   - Check if API key matches between frontend and backend
   - Ensure backend is running (check health endpoint)

6. **Migration Failures**
   - If `alembic upgrade head` fails with async driver error:
     - Set `RUN_MIGRATIONS=false` temporarily
     - Manually run migrations after backend starts:
       ```bash
       railway run python -c "from src.db.session import engine; from src.db.models import Base; import asyncio; async def run(): async with engine.begin() as conn: await conn.run_sync(Base.metadata.create_all); asyncio.run(run())"
       ```

7. **Gunicorn Not Found Error**
   - **Error**: `./start_production.sh: line 61: exec: gunicorn: not found`
   - **Cause**: Gunicorn might not be properly installed in Docker multi-stage build
   - **Solution**: The application automatically falls back to Uvicorn if gunicorn is not found
   - To ensure gunicorn is installed, rebuild the Docker image with updated Dockerfile

8. **ML Model Not Found Warning**
   - **Warning**: `Warning: ML model not found at src/artifact/model.pkl`
   - **Cause**: Model files excluded by .dockerignore
   - **Solution**: Updated .dockerignore to include model files. Ensure model.pkl, model_artifact_v1.pkl, and preprocessor.pkl are in src/artifact/ directory
   - If models are missing, the application will start but prediction features will fail
